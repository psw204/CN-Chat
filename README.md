# 실시간 채팅 애플리케이션 - 프로젝트 (컴퓨터 네트워크)

## 프로젝트 개요

본 프로젝트는 **Django** 기반 백엔드와 **React** 기반 프론트엔드로 구성된 **실시간 채팅 애플리케이션**을 구축하는 것을 목표로 합니다. 사용자는 회원가입 후 로그인하여 다른 사용자와 실시간으로 텍스트와 이미지 메시지를 주고받을 수 있으며, 단체 채팅방 및 파일 공유 기능도 지원합니다. 또한, **Django Channels**를 활용한 **WebSocket**을 통해 실시간 메시징 기능을 제공합니다.

## 주요 기능

* 사용자 인증 시스템 (회원가입, 로그인, JWT 토큰 기반 인증)
* 1:1 실시간 채팅 기능 (텍스트 및 이미지 메시지)
* 단체 채팅방 기능
* 파일 전송 기능 (문서, 오디오)
* 온라인/오프라인 상태 관리
* 사용자 차단/해제 기능
* 채팅방에서 나가기 기능

## 기술 스택

* 백엔드: **Django**, **Django REST Framework**, **Django Channels** (WebSocket)
* 프론트엔드: **React**
* 데이터베이스: **SQLite** (로컬 개발용)
* 인증 시스템: **JWT** 토큰 기반 인증

## 실행 방법

### 필수 조건

* `Node.js`와 `npm`이 설치되어 있어야 합니다.
* `Python` 및 `Django`가 설치되어 있어야 합니다.
* `Django` 프로젝트의 가상 환경을 활성화해야 합니다.

### 1. 프로젝트 클론

먼저, 프로젝트를 클론합니다.

```bash
git clone https://github.com/yourusername/yourproject.git
cd yourproject
```

### 2. 백엔드 (Django) 설정

1. 가상 환경을 생성하고 활성화합니다.

   ```bash
   python -m venv venv
   venv\Scripts\activate  # Windows
   source venv/bin/activate  # MacOS / Linux
   ```

2. 필요한 Python 패키지를 설치합니다.

   ```bash
   pip install -r backend/requirements.txt
   ```

3. 데이터베이스 마이그레이션을 적용합니다.

   ```bash
   python manage.py migrate
   ```

4. Django 서버를 실행합니다.

   ```bash
   python manage.py runserver
   ```

### 3. 프론트엔드 (React) 설정

1. `frontend` 폴더로 이동하여 필요한 패키지를 설치합니다.

   ```bash
   cd frontend
   npm install
   ```

2. 개발 서버를 실행합니다.

   ```bash
   npm run dev
   ```

### 4. 전체 실행

**프론트엔드**와 **백엔드**를 동시에 실행하려면, 프로젝트의 루트 디렉터리에서 `npm run dev` 명령을 사용합니다.

```bash
npm run dev
```

이 명령은 `frontend`와 `backend` 서버를 동시에 실행합니다.

### 5. 추가 설정

* JWT 토큰을 발급받으려면 로그인 기능을 사용하여 로그인 후 토큰을 받습니다.
* 채팅 기능을 사용하려면 다른 사용자와 함께 채팅방을 생성하고 실시간 메시지를 교환합니다.

## 주요 기능 설명

### 사용자 인증

* **회원가입**: 사용자는 이메일, 닉네임, 비밀번호, 프로필 사진을 통해 회원가입합니다.
* **로그인**: 이메일/비밀번호를 통해 JWT 토큰을 발급받습니다.
* **인증 상태 유지**: 발급된 토큰은 로컬 스토리지에 저장되며, API 요청 시 헤더에 포함됩니다.

### 실시간 채팅

* WebSocket을 사용하여 텍스트 및 이미지를 실시간으로 주고받습니다.
* 사용자는 1:1 채팅방을 생성하고, 이전 메시지 이력을 로드할 수 있습니다.

### 파일 전송

* 사용자는 채팅방에서 파일을 전송할 수 있으며, 서버를 통해 저장된 파일은 상대방이 다운로드할 수 있습니다.

### 온라인/오프라인 상태

* 사용자는 로그인하면 온라인 상태로 전환되며, 로그아웃하면 오프라인 상태로 전환됩니다.
* 차단된 사용자는 항상 오프라인으로 표시됩니다.

### 차단/해제

* 사용자는 다른 사용자를 차단하거나 해제할 수 있으며, 차단된 사용자는 더 이상 메시지를 주고받을 수 없습니다.
